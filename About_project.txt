1) реализуется полный процесс работы c MNSIT так, как буд-то необходимо решить задачу в рамках kaggle
2) цель: код пишется так, чтобы его можно было масшатабировать на другие датасеты
3) в данный момент код довольно костыльный и такой масштабируемости нету
4) не производится менеджмента ресурсов во время выполнения скриптов
5) код ревью и тестирование не производились, защитных механизмов внутри функций нет
6) в качетве обозначения namespace используется просто префикст к функции , например, __namespace__func()
7) префиксы в названиях файлах, например, 1_step_file.py обозначают не просто последовательность выполнения скриптов,
   а логические шаги при работе с очередной задачей в рамках ML competition
8) *специфичные названия функций, например load_all_public_images_ana_labels, мотивируются следующим образом:
    1) public_data = картинки и метки из train MNIST аналогично kaggle разметки датасета
    2) private_data =картинки и метки из test MNIST аналогично kaggle разметки датасета
   * название execution_of_existing_functions в конкретном namespace мотивируется тем, что по сути, эти функции просто
   запускают уже реализованный функционал из своего namespace в нужной последовательности
9) training loop - файл который можно запускать многократно
